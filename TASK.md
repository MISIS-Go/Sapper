# Проект “Sapper”
## Описание проекта
Реализовать игру в сапера. Проект должен содержать стартовое меню и окно игры. При закрытии окна игры прогресс должен сохраняться на компьютере в файловую систему. При выполнении работы проверяйте, соблюдаете ли вы пункты в критериях оценивания задания на Moodle.


### Базовая часть (5 баллов):
- [x] Создать интерфейс, описывающий свойства клетки.
- [x] Абстрактный класс клетки, реализующий интерфейс и добавляющий необходимые
свойства и методы.
- [x] Классы-наследники: мина, флаг, ячейка с числом, пустая ячейка (вокруг которой нет мин).
- [x] Класс поля со свойствами для размера: количества ячеек по горизонтали и вертикали.
- [x] Количество мин должно быть в диапазоне от 20% до 40% всех ячеек.
- [x] Если игрок попал на мину, то игра заканчивается проигрышем.
- [x] Стартовое меню с 2-мя кнопками: “Новая игра” и “Продолжить игру”. 
- [x] Новая игра начинается с пустого поля 5х5, а продолжение игры - с сохраненного поля.
- [x] В новом окне должно открываться игровое поле с размеченными клетками. 
- [x] Управление мышью: левая кнопка - активировать поле, правая - поставить флаг.
- [x] Создать класс для сериализации информации в формате JSON. 
- [x] Сохранение игры осуществлять при закрытии окна игры.


### Дополнительная часть (3 балла):
- [x] Создать и подключить к проекту библиотеку “Model”. 
- [x] В неё поместить все файлы, отвечающие за бизнес-логику вашей игры.
- [x]  Классы, отвечающие за визуальную составляющую и взаимодействие с игроками оставить в основном проекте “Sapper”.
- [x] Для класса, отвечающего за логику игры добавить модификатор partial. 
- [x] Создать ещё одну часть этого класса partial для реализации победы, если все мины закрыты флагами, а остальные ячейки поля активированы.
- [x] При генерации клеток учесть, чтобы каждую мину окружало не менее 3 клеток с числами (иначе углы или другие части могут быть “заблокированы”).
- [x] В главном меню добавить поле для выбора папки для сохранения игры.
- [x] В главном меню добавить возможность выбора размера поля (5х5, 10х10, 5х10 и т п).
- [x] Масштабировать размер ячеек и поля при создании новой игры в зависимости от установленного размера. (Можно сделать списком, слайдером или текстовыми полями с ограничениями.)
